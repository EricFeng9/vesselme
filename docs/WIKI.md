# VesselMe WIKI

## 已实现功能索引（v0.1 MVP）

### 1. 项目与文件管理
- 支持打开目录并识别图像格式：`png/jpg/jpeg/tif/tiff`。
- 左侧文件列表显示每张图像的标注状态（`labeled/unlabeled`）与修改状态（`dirty/clean`）。
- 切图时自动读取同级目录下满足规则 `<image_stem>_[<label_name>].tar` 的标注包。
- 自动加载时会校验 `meta.json.image_name` 与当前图片一致，不一致则拒绝。
- 支持左右方向键切换上一张/下一张图像。

### 2. 标签系统
- 支持当前图像多标签实例管理：创建、选择、重命名、删除、导入。
- 标签字段在内存中维护：`label_name/display_color/visible/locked/created_at/updated_at`。
- 标签列表支持锁定状态切换：`[L]`（锁定）/`[U]`（可编辑）。
- 锁定标签后禁止修改像素：禁画、禁清空、禁撤销、禁重做。
- 标签重命名后会同步更新：
  - 磁盘 tar 文件名（如果原文件已存在）；
  - 后续保存时 `meta.json.label_name`。
- 删除标签时弹确认框，确认后删除内存标签和对应磁盘 `.tar`。
- 新建标签默认名 `label1, label2...`，默认颜色白色。
- 手工导入 `.tar` 仅写入内存（标记 dirty），需要用户主动保存后才落盘。

### 3. 画布与标注
- 中央画布支持图像层与掩码叠加层实时显示。
- 掩码内存结构：`numpy.ndarray(H, W, dtype=uint8)`。
- 掩码值统一二值：背景 `0`、前景（血管）`255`。
- 工具与交互：
  - 画笔（`B`）
  - 橡皮擦（`E` 或 `Shift + 左键`）
  - `Ctrl + 左键拖动` 调整画笔大小
  - `中键拖动` 或 `Space + 左键` 平移
  - `右键拖动` 缩放（灵敏度可调）
  - 鼠标滚轮缩放
  - `A` 显示/隐藏标注层
  - `[` / `]` 缩小/增大画笔
- 支持适应窗口和 1:1 视图。
- 显示笔刷圆形预览。

### 4. 编辑历史
- 支持撤销/重做（`Ctrl+Z` / `Ctrl+Y`）。
- 一次按下到抬起笔画计为一步历史。
- 历史容量默认 100（满足至少 20 步要求）。

### 5. 保存与导出
- `Ctrl+S` 或 `S` 保存当前正在编辑标签。
- 保存路径固定为当前图片目录，文件名固定为：
  - `<image_stem>_[<label_name>].tar`
- `.tar` 固定包含：
  - `mask.npy`（uint8 二值单通道）
  - `meta.json`（标签元数据）
- `meta.json` 包含 `visible/locked`，可恢复标签显示与锁定状态。
- 支持导出黑色背景 PNG 叠加图：
  - `<image_stem>_[<label_name>]_stroke.png`
  - 非标注区域为纯黑 `(0,0,0)`，标注区域为标签颜色。
- 切图/退出时，若当前图存在未保存标签，弹窗提供 `Save / Discard / Cancel`。

### 6. GUI 结构
- 主界面布局：
  - 左：文件列表
  - 中：主画布
  - 右：标签列表与标签操作区
  - 顶：菜单栏
  - 左竖工具栏：画笔、橡皮、清空
  - 底：状态栏（文件名、缩放、画笔、工具、标签、保存状态）
- 右侧包含右键缩放灵敏度调节控件，设置持久化到 `~/.vesselme_config.json`。

### 7. 交互与视觉升级（Labelme 风格靠拢）
- 恢复显式“打开目录”入口：
  - 左侧文件面板新增 `Open Folder` 主按钮；

- 顶部主工具栏重构为高频动作入口：打开目录、保存、导入、导出、上一张、下一张。
- 三栏布局权重重设（文件/画布/标签），并加面板边界与分区标题，降低信息噪音。
- 工具栏按钮增加选中态（Brush/Eraser），当前工具一眼可见。
- 引入统一浅色主题（面板、按钮、列表、状态栏），提升可读性与视觉一致性。
- 增加文件计数信息（`N images`）和画布操作提示（缩放/平移提示）。
- 画布空态提示升级，直接展示启动指引与核心快捷键。
- 画布鼠标光标跟随工具状态变化（画笔/橡皮十字光标、平移抓手、锁定状态箭头）。

### 8. 目录扫描与面板拖拽修复
- 打开目录时，图片扫描从“单层”升级为“递归子目录扫描”，会加载所选目录下全部层级中的 `png/jpg/jpeg/tif/tiff`。
- 文件列表显示改为“相对于所选根目录的路径”，避免不同子目录同名图片混淆。
- 三栏 `QSplitter` 调整为不可折叠子面板，拖拽时不再把某列压缩到消失。
- 为三栏设置最小宽度（文件/画布/标签），拖拽行为改为稳定的弹性宽度调节。

### 9. 画笔与标签列表交互优化
- 新增前端画笔大小控制区：`Slider + 数值输入框`，二者与当前画笔大小双向同步。
- 新增 `Ctrl + 鼠标滚轮` 快捷调节画笔大小（每次滚动步长 1，范围 `1~512`）。
- 打开图片后不再自动创建 `label1`，标签列表默认可为空；无标签时画布仅预览图像，绘制会提示先创建/选择标签。
- 标签列表移除 `[V] [U]` 缩写展示，改为主名称 + 可读状态后缀（如 `hidden/locked`）。
- 标签项左侧新增圆形颜色色块，点击色块可直接弹出颜色选择器修改该标签颜色。

### 10. 回归修复（画笔尺寸与标签行重叠）
- 修复 `Ctrl + 滚轮` 调节画笔后“光标变了但实际笔画不变”的问题：
  - 统一画笔尺寸更新入口为 `CanvasWidget.set_brush_size()`；
  - 滚轮快捷键、`[` `]`、面板滑块/输入框全部走同一入口；
  - 绘制半径改为与当前画笔尺寸一致的统一半径计算。
- 修复标签列表文本重叠问题：
  - 去除 `QListWidgetItem` 自身文本渲染；
  - 标签行仅使用自定义 widget（颜色圆点 + 标签名）渲染，避免重复绘制导致糊在一起。

### 11. 标签行内图标交互重构
- 将标签 `visible/lock` 操作从下方按钮区迁移到每一行标签右侧图标：
  - 眼睛图标：显示/隐藏标签；
  - 锁图标：锁定/解锁标签；
  - 颜色圆点：修改该标签颜色。
- 删除下方操作区的 `Color / Toggle Visible / Toggle Lock` 三个按钮，减少冗余操作入口。
- 标签主操作保留：`New / Import / Rename / Delete`，并保留快捷键与菜单行为（例如 `A` 仍可切换当前标签可见性）。

### 12. 标签行图标渲染修复与扩展
- 行内图标由 emoji 改为本地绘制矢量 icon（`QPainter`），避免字体缺失或显示裁切。
- 新增标签行内 `Rename` 与 `Delete` 小图标，支持直接按行重命名/删除。
- 下方操作区 `Rename / Delete` 大按钮继续保留，形成“行内快捷 + 面板主操作”双入口。
- 调整标签行高度与图标尺寸（20px 按钮、14px 图标）以确保图标完整显示。

### 13. 图标体系精简与控件收敛
- 标签行图标改为“系统主题 icon 优先，失败则使用简洁矢量 fallback”，整体风格更轻量一致。
- 图标按钮改为透明背景 + hover 高亮，减少胶囊边框噪声，避免“图标被框挤压”的观感。
- 标签行尺寸与图标尺寸重新标定（行高 38、按钮 22、图标 16）以避免裁切。
- 根据 UI 收敛要求，移除下方操作区 `Rename / Delete` 按钮，仅保留标签行内同等操作入口。

### 14. 图标迭代与面板细化
- 按需替换标签行 `眼睛` 与 `改名` 图标：优先使用系统主题轻量 icon，缺失时使用新 fallback 绘制。
- 缩小标签名前颜色圆盘尺寸至 14px，并分离专用样式，避免被通用图标按钮尺寸规则放大。
- 标签面板下方“当前标签名称显示”控件已从布局中移除，降低视觉噪声。

### 15. 标签行图标风格统一
- 移除“主题图标优先”的混用策略，统一改为本地矢量线性图标渲染。
- `eye / lock / rename / delete` 全部使用同一套线宽、圆角、笔触规范，确保视觉一致。
- 解决不同系统主题下图标风格漂移问题（同一版本跨机器观感一致）。

### 16. 最小笔刷精度修复
- 修复 `brush size = 1` 仍绘制多像素的问题。
- 新规则：
  - `size <= 1` 时仅修改单个目标像素（严格 1px）；
  - `size > 1` 时使用非抗锯齿栅格绘制，避免边缘扩散导致笔触超预期。

### 17. 低档笔刷精细映射
- 进一步修复低档位跳变问题（`1px` 太小、`2px` 过大）：
  - `1px`：单像素；
  - `2px`：严格 `2x2` 方形 footprint；
  - `3px`：严格 `3x3` 方形 footprint；
  - `>=4px`：保持圆形笔刷（非抗锯齿）用于中大尺寸连续涂抹。
- 目标是让血管精细标注在低尺寸段有可预测、可控的增量。

### 18. 绘图算法重写（中心对齐与连续路径）
- 将“按事件点直接落笔”重写为“连续路径采样 + 子像素圆形盖章”：
  - 鼠标按下后记录起点，拖动时对线段按 `<=0.5px` 间距采样；
  - 每个采样点都按当前笔刷半径盖章，保证细线不断裂。
- 统一“预览光标中心”和“实际落笔中心”坐标体系（浮点图像坐标），修复中心偏移感。
- 笔刷 footprint 由像素中心判定（点到圆距离），最小半径 `0.5`，确保 `1px` 行为稳定可控。

### 19. 半像素笔刷档位与快捷调节
- 笔刷范围改为 `0.5px ~ 50.0px`，支持 `0.5px` 步进。
- 右侧控件更新为半像素体系：
  - 滑块：`1~100` 映射到 `0.5~50.0`；
  - 输入框：`0.5` 步进、1 位小数。
- `Ctrl + 鼠标滚轮` 快捷调节规则：
  - `<=10px` 区间按 `0.5px` 调整；
  - `>10px` 区间按 `1px` 调整。

### 20. 标签颜色渲染为纯色
- 标注叠加层 alpha 从半透明改为不透明（`255`），按标签颜色实心渲染。

### 21. 右键橡皮擦与渲染优化

### 31. 从图片导入 mask（Import from image）
- 顶部主工具栏 `Import .tar` 旁新增 `Import from image` 按钮，并在 `File` 菜单增加同名导入入口。
- 导入流程先弹三选一确认框：`Create new label` / `Overwrite current label` / `Cancel`。
- 新建模式新增可选自定义弹窗：可输入标签名并可选选择渲染颜色；也可直接使用默认 `labelN + 白色`。
- 覆盖模式会校验当前是否选中标签；无选中时拒绝并提示。
- 选择图片后执行统一预处理：
  - 仅读取灰度通道；
  - 先按阈值 `>=128` 二值化为 `0/255`（避免低灰度噪声导致血管变粗）；
  - 在二值图上按当前图像尺寸做最近邻 `resize`；
  - 转换为程序标准 `numpy.ndarray(H, W, dtype=uint8)` 二值 mask。
- 导入结果仅更新内存标签并立即渲染到画布；不会立刻写入磁盘，需用户点击 `Save` 才生成/覆盖 `.tar`。

### 32. 画布性能优化（大面积掩码卡顿修复）
- 修复根因 1：移除 `paintEvent` 中每帧“全图重建 overlay pixmap”的路径，改为持久化缓存 `overlay_rgba + QImage`。
- 修复根因 2：绘制过程中不再对每个采样点触发 `dirtyChanged`；改为“一次完整笔画结束后”仅触发一次。
- 新增局部更新机制：每次笔刷仅更新受影响区域的 overlay 缓存，而不是重算整张图。
- `undo/redo` 后会自动重建 overlay 缓存，保证显示与 mask 始终一致。
- 结果：导入大掩码后继续擦除/绘制时，UI 列表刷新频率和叠加层重建开销显著下降。

## 代码结构
- `main.py`：启动入口。
- `vesselme/main.py`：应用运行函数。
- `vesselme/ui/main_window.py`：主窗口、布局和业务事件绑定。
- `vesselme/ui/canvas_widget.py`：画布渲染、缩放平移、绘制与历史记录接入。
- `vesselme/core/history.py`：笔画历史栈（undo/redo）。
- `vesselme/data/models.py`：核心数据模型与约束。
- `vesselme/data/io.py`：`.tar` 标注包读写与透明导出。
- `vesselme/data/settings.py`：本地设置加载与保存（右键缩放灵敏度）。
- `vesselme/services/project_service.py`：项目目录扫描、图像加载、自动标签发现。
- `vesselme/services/label_service.py`：标签生命周期与保存导出服务。
- `vesselme/utils/qt.py`：Qt 图像转换工具函数。
