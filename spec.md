# 眼底图像血管分割标注工具开发规范（spec）

版本：`v0.1`  
目标：先复刻 `pair` 的核心操作逻辑，再吸收 `labelme` 的简洁 GUI 与工程化思想，开发一个专为眼底血管分割设计的本地标注工具。

## 1. 项目目标与范围

### 1.1 主要目标
- 支持眼底图像逐张标注血管分割掩码（mask）。
- 操作逻辑尽量与 `pair` 保持一致，降低迁移学习成本。
- 界面风格参考 `labelme`：简洁、清晰、低学习成本。
- 输出可直接用于语义分割训练的数据格式。

### 1.2 非目标（首版不做）
- 不做云端协作与账号系统。
- 不做 3D 标注与多模态标注。
- 不内置复杂自动分割模型。

## 2. 用户工作流（Pair 兼容）

1. 启动软件并建议最大化窗口。  
2. 打开待标注文件夹，批量加载图像。    
3. 软件自动扫描当前图片同级目录，加载符合命名规则 `图片名_[标签名].tar` 的标签文件。  
4. 可手动导入任意“合规 tar 标签文件”，导入后立即在 GUI 中显示标签名、标签颜色与标注内容，但仅写入内存。  
5. 为当前图像新建标签（默认名 `label1`，默认显示颜色白色，可修改），或选择已有标签进行编辑。  
6. 选中当前标签后，使用左侧画笔工具对眼底图像血管进行标注。  
7. 标注像素在内部统一保存为二值：背景 `0`，血管 `255`；标签颜色仅用于 GUI 渲染显示。  
8. 使用快捷操作完成高效编辑：  
   - `B`：画笔  
   - `E`：橡皮  
   - `Ctrl + 鼠标滚轮`：调整画笔大小  
   - `鼠标右键`：临时橡皮  
   - `中键拖动` 或 `Space + 左键`：平移画布  
   - `滚轮`：缩放图像  
   - `Ctrl+Z / Ctrl+Y`：撤销 / 重做  
   - `A`：显示 / 隐藏标注层  
   - `[` / `]`：缩小 / 增大画笔  
   - `1~9`：切换标签  
   - `S` 或 `Ctrl+S`：保存当前标签  
   - `← / →`：切换上一张 / 下一张图片  
9. 点击保存按钮或 `Ctrl+S` 保存“当前正在编辑的标签”，将内存状态写回对应 `.tar` 文件。  
10. 可以通过按钮或者键盘的左右方向键切换目录内下一张图像继续标注。

## 3. 功能需求（MVP）

### 3.1 文件与项目管理
- 打开文件夹并识别图像格式：`png/jpg/jpeg/tif/tiff`。
- 左侧文件列表显示缩略图、标注状态、是否已修改。
- 支持断点续标：打开图片时自动扫描并加载同级目录下对应 `图片名_[标签名].tar` 文件。
- 默认不自动写盘，避免覆盖未确认的导入或编辑结果。
- 支持以 `.tar` 作为标注主存储包格式，包内使用 `numpy`（`mask.npy`）保存像素标注。

### 3.2 标签系统
- 不再依赖全局 `ColorTable.json` 作为必需输入（可保留为可选模板功能）。
- 每张图可以新建多个“标签实例”（例如 `label1`, `label2`）。
- 标签字段：`label_name`, `display_color`, `visible`, `locked`, `created_at`。
- 标签颜色仅影响前端渲染样式，不影响落盘标注值。
- 当前版本血管任务统一二值编码：背景 `0`、血管 `255`。
- 标签列表支持：显示/隐藏预览、重命名、删除。
- 标签重命名并保存后，必须同步更新：
  - 标签对应 `.tar` 文件名；
  - `.tar/meta.json` 内 `label_name` 字段。
- 删除标签时必须弹确认框；确认后删除内存标签并删除磁盘对应 `.tar` 文件。

### 3.3 画布与标注
- 图像层 + 标注层叠加显示。
- 工具：画笔、橡皮擦、清空当前标签区域。
- 参数：画笔直径、颜色。
- 视图控制：缩放、平移、适应窗口、1:1。
- 显示笔刷圆形预览及当前标签信息。
- 标注层在内存中以 `numpy.ndarray(H, W, dtype=uint8)` 维护，并实时可视化叠加到原图。

### 3.4 编辑历史
- 撤销/重做（至少 20 步）。
- 一次连续笔画（按下到抬起）记为一步。
- 未保存离开时提示。

### 3.5 数据导出
- 保存单位为“单图像-单标签”。
- 每个标签保存为一个 `.tar` 文件（标准交换格式）。
- `.tar` 内固定包含两个文件：
  - `mask.npy`：单通道 `uint8` 二值标注（0/255）
  - `meta.json`：标签名称与 GUI 颜色等元数据
- 可选导出叠加预览图（overlay）。
- 预留 COCO/RLE 导出接口（非 MVP 必做）。
- 支持导出黑色背景 PNG：背景为纯黑，标注区域按标签颜色渲染。

### 3.6 保存交互规范
- 点击“保存”按钮或按 `Ctrl+S` 保存“当前正在编辑的标签”。
- 保存路径与文件名不可由用户自定义，必须使用固定规则：
  - 保存目录：当前图片所在目录；
  - 保存文件名：`<图片名>_[<标签名>].tar`（示例：`18_A_test_[label1].tar`）。
- 切换标签后，保存目标自动切换为新选中的标签文件。
- 导入标签后仅加载到内存，必须由用户主动保存后才写盘。
- 若当前标签存在未保存修改，切图/退出前需弹窗确认（保存/不保存/取消）。

### 3.7 导入与自动加载规范
- 手动导入：用户为当前图片选择一个 `.tar` 文件导入。
- 导入校验通过后，立即将 `meta.json + mask.npy` 渲染到 GUI，并加入当前图片标签列表（内存态）。
- 支持从普通图片导入 mask（`import from image`）：
  - 先弹确认框：`新建 label 导入` / `覆盖当前选中 label` / `取消`；
  - 新建模式可选弹窗：自定义 `label_name` 与渲染颜色（可跳过，默认 `labelN + 白色`）；
  - 选择图片后，仅取灰度通道并先做二值化（默认阈值 `>=128` 为前景）；
  - 若尺寸不一致，按当前图像尺寸进行最近邻 `resize`（在二值图上操作）；
  - 导入结果仅更新内存态标签，必须由用户主动点击保存后才写入 `.tar`。
- 覆盖模式必须校验“当前是否有选中标签”，无选中标签时拒绝覆盖并提示用户。
- 自动加载：每次打开一张图片时，自动扫描其同级目录下符合命名规则的标签 `.tar` 并载入。
- 若 `.tar` 内 `image_name` 与当前图片不匹配，默认拒绝加载并提示错误。

## 4. 界面设计规范（参考 Labelme）

### 4.1 主布局
- 左侧：文件列表（缩略图 + 状态）。
- 中央：主画布（图像 + 标注）。
- 右侧：标签表（ID/颜色/描述）。
- 顶部：菜单栏 + 主工具栏。
- 左竖栏：画笔、橡皮、平移、缩放等工具。

### 4.2 视觉与交互
- 主题简洁，支持深色模式（可选）。
- 工具按钮显示快捷键提示（tooltip）。
- 状态栏实时显示：文件名、缩放倍率、画笔大小、标签名、保存状态。

### 4.3 异常反馈
- 文件读取失败、标签格式错误、保存失败均需给出明确弹窗提示。
- 非阻塞错误（如某张图读失败）记录到日志并允许继续标注。

## 5. 数据规范

### 5.1 示例目录结构
```text
image1/
  xxx.png
  xxx_[label1].tar
  xxx_[label2].tar
  exports/                  # 可选导出目录
    xxx_[label1].png
```

### 5.2 标注 tar 内容规范
```json
{
  "image_name": "xxx.png",
  "label_name": "label1",
  "display_color": [255, 255, 255],
  "mask_encoding": "uint8_binary",
  "foreground_value": 255,
  "background_value": 0,
  "mask_file": "mask.npy",
  "created_at": "2026-04-13T12:00:00+08:00",
  "updated_at": "2026-04-13T12:10:00+08:00",
  "tool_version": "v0.1"
}
```

`.tar` 目录内容固定为：
- `mask.npy`
- `meta.json`

### 5.3 mask 编码约定
- 类型：`uint8` 单通道图。
- 默认取值：背景 `0`，血管 `255`。
- 内存结构：`numpy.ndarray`，形状 `H x W`，`dtype=uint8`。
- 标签渲染颜色不参与 `mask.npy` 的像素值存储。
- 标准保存映射：`<image>_[<label_name>].tar`。
- 黑底导出映射：`<image>_[<label_name>]_stroke.png`。

### 5.4 文件命名与匹配规则
- 图片文件：`<image_name>.<ext>`
- 标签文件：`<image_name>_[<label_name>].tar`
- 自动扫描时只匹配当前 `image_name` 前缀的 `.tar` 文件。
- `label_name` 禁止使用文件系统非法字符（如 `\\ / : * ? " < > |`）。

### 5.5 为什么主存储使用 numpy 而不是 JSON
- 画笔标注属于像素级栅格数据，JSON 不适合存储大量逐像素结果（体积大、读写慢）。
- `numpy` 数组可直接用于实时绘制、撤销重做和训练管线，读写效率更高。
- JSON 仅用于元数据（如标签名称、GUI颜色、时间戳、版本信息）。

## 6. 技术方案

### 6.1 技术栈
- GUI：`PySide6`（推荐）或 `PyQt5`
- 图像处理：`numpy`, `opencv-python`, `Pillow`
- 配置解析：`json`（可选 `pydantic` 进行校验）
- 打包：`PyInstaller`

### 6.2 分层架构
- `ui`：窗口、控件、事件绑定
- `core`：画布引擎、笔刷逻辑、命令栈、坐标变换
- `data`：项目配置、标签配置、文件读写
- `services`：自动保存、缩略图缓存、导出
- `plugins`：后续扩展（半自动分割等）

### 6.3 关键技术点
- 精准坐标映射：窗口坐标 -> 画布坐标 -> 图像像素坐标。
- 连续笔画插值，避免快速拖动产生断线。
- 撤销重做采用命令模式，记录 patch 减少内存占用。
- 大图性能优化：局部重绘、缓存缩放结果、异步缩略图。
